# Portfolio Optimization

This project explores numerical optimization methods applied to portfolio
construction, from least-squares problems to Markowitz mean-variance
optimization and higher-moment (kurtosis-aware) portfolio selection. It is
derived from an academic optimization project at **ENSTA Paris**.

**For the methodology, results, and discussion, see
[`Markowitz_portfolio_optimization_report.pdf`](Markowitz_portfolio_optimization_report.pdf).**

## Contents

- `scripts/` — all the Python scripts (see below).
- `figures/` — plots and figures generated by the scripts.
- `Markowitz_portfolio_optimization_report.pdf` — the written report
  summarizing methodology and results.

### Scripts

The numbered scripts follow the same order as the questions in the project
report, so running them `01` through `10` retraces the project end to end:

- `01-cvxpy-least-squares.py` — solves a bounded least-squares problem with
  CVXPY and benchmarks solve time as problem size grows.
- `02-slsqp-least-squares.py` — same least-squares problem solved with SciPy's
  SLSQP, for comparison against the CVXPY approach.
- `03-markowitz-portfolio.py` — classic Markowitz mean-variance portfolio,
  evaluated for varying numbers of assets.
- `04-markowitz-portfolio-probabilistic.py` — a probabilistic
  (chance-constrained) variant of the Markowitz portfolio.
- `05-asset-time-series-plot.py` — plots cumulative returns of the assets over
  the training and testing periods.
- `06-montecarlo-portfolio-visualization.py` — visualizes random portfolios
  via Monte Carlo sampling alongside the computed optimal portfolios.
- `07-sqp-vs-slsqp.py` — compares a custom SQP solver against SciPy's SLSQP on
  a non-convex problem (speed, iterations, constraint satisfaction).
- `08-kurtosis-scp-convergence.py` — solves the kurtosis-minimization problem
  with Sequential Convex Programming (SCP) and studies convergence from
  different starting points.
- `09-kurtosis-methods-comparison.py` — compares SLSQP, SCP, and Markowitz on
  the kurtosis-aware portfolio problem.
- `10-kurtosis-sdp.py` — formulates and solves the kurtosis problem as a
  semidefinite program (SDP) with CVXPY.

Two shared modules are imported by the numbered scripts above and are not
meant to be run on their own, which is why they carry no number:

- `Supporting_functions.py` — shared utilities: downloading market data (via
  `yfinance`), computing return statistics, Monte Carlo portfolio sampling,
  plotting helpers, and the core portfolio-optimization routines (Markowitz,
  kurtosis-based SLSQP and SCP solvers).
- `sqp_solver.py` — a from-scratch Sequential Quadratic Programming (SQP)
  solver applied to a non-convex constrained problem, used by
  `07-sqp-vs-slsqp.py`. (It keeps an underscore instead of a numeric prefix
  because it is imported as a module — `from sqp_solver import *` — and a
  name starting with a digit or containing a hyphen isn't a valid Python
  module name.)

## Reproducibility

Run the scripts in numeric order from inside the `scripts/` directory, e.g.:

```bash
cd scripts
python 01-cvxpy-least-squares.py
python 02-slsqp-least-squares.py
...
python 10-kurtosis-sdp.py
```

Running them in order and from that directory keeps the shared-module imports
working and reproduces the project's results, printed metrics, and figures
step by step, in the same sequence as the report. A few things to know before
running them:

- **Internet access is required.** Every script pulls historical price data
  live from Yahoo Finance through `yfinance`; results depend on that data
  still being available and unchanged.
- **Some scripts open plot windows and wait.** `05`, `08`, and `09` call
  `plt.show()`, which blocks until you close the figure window. Run them in
  an environment with a display, or comment out `plt.show()` for a
  fully non-interactive/headless run.
- **`sqp_solver.py` runs on import, not just on call.** It's the original
  standalone script kept as a shared module, so its top-level code (running
  the SQP solver and regenerating a plot) executes again whenever
  `07-sqp-vs-slsqp.py` does `from sqp_solver import *`. This is expected, not
  a bug — just don't be surprised to see it print/plot twice.
- Randomized steps (e.g. the SQP initial guess, Monte Carlo sampling) use
  fixed seeds (`np.random.seed(1)` / `RandomState(seed=123)`), so numerical
  results should match run to run given the same input data.

## Requirements

Python 3, with `numpy`, `pandas`, `matplotlib`, `scipy`, `cvxpy`, `yfinance`
(historical stock price data), and `riskfolio-lib` (portfolio composition
pie charts, used by scripts `06` and `09`).
